Git e GitHub - Anotações Básicas

O que é Git e GitHub?

Git é um sistema de controle de versão distribuído, que serve para registrar o histórico de alterações em arquivos (como códigos, documentos, etc.).
Ele Salva diferentes versões de um projeto, e volta para versões anteriores se algo der errado.

GitHub é uma plataforma online que hospeda repositórios Git, ou seja, os salva na nuvem.

Conceitos Básicos

Abaixo temos uma lista de conceitos muito importantes e que são bastante utilizados nas ferramentas Git e GitHub

  • Repositório (repo) - É o diretório onde seu projeto é armazenado. Pode ser local (no computador) ou remoto (no GitHub).
  • Commit - É um “salvamento” de alterações com uma descrição do que foi feito.
  • Branch - Uma “linha” de desenvolvimento separada da principal (usado para testar novas ideias sem afetar o projeto principal).
  • Merge - Junta as mudanças de uma branch com outra.
  • Clone - Copia um repositório remoto para seu computador.
  • Push - Envia os commits locais para o GitHub.
  • Pull - Baixa as alterações do GitHub para seu computador.
  • Pull Resquest - É uma solicitação feita no GitHub para adicionar (merge) uma nova funcionalidade (branch).
  • Stage/Staging Area - Área onde você prepara os arquivos antes de fazer o commit.
  • Git ignore - Usado para ignorar arquivos, eles não são enviados para o GitHub.
